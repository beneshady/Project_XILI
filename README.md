# 肉鸽象棋 (Chess Roguelike)

一款融合《Shotgun King》回合制逻辑与中国象棋规则的肉鸽生存游戏。

## 🎮 游戏介绍

这是一款极简风格的策略肉鸽游戏，在 8x8 的中国象棋棋盘上，你需要控制王（玩家）在不断逼近的敌人中生存下去。

### 核心玩法
- **回合制**：每回合你移动一步，然后所有敌人向你逼近一步
- **九宫格移动**：王可以向周围 8 个方向（包括斜向）移动一格
- **威胁系统**：可以看到所有敌人的威胁范围（红色），必须躲避
- **击杀机制**：直接踩到敌人可以击杀并获得分数
- **无限模式**：没有固定目标，尽可能存活更多回合

### 实体说明

| 棋子 | 外观 | 行为 | 威胁范围 |
|-------|------|------|-----------|
| **王（玩家）** | ♔ 蓝色方块 | 可向九宫格内任意方向移动 | - |
| **兵（经验包）** | ♟ 绿色圆形 | 每次向你移动一格 | 向你方向一格 |
| **车（强敌）** | ♜ 珊瑚色方块 | 按象棋规则移动 | 十字方向，无限距离 |
| **马（强敌）** | ♞ 橙色方块 | 按象棋规则移动 | 日字跳跃，8 个位置 |

### 敌人生成规则
- **第 1 回合**：无初始敌人
- **每回合生成**：1-2 个兵
- **第 5 回合起**：50% 概率生成马
- **第 10 回合起**：50% 概率生成车

## 💻 技术栈

- **渲染引擎**：HTML5 Canvas
- **编程语言**：纯 JavaScript (ES6+)
- **开发语言**：TypeScript（带类型定义）
- **外部依赖**：无（零依赖，无需 npm/yarn）

## 🏗️ 架构设计

详细的架构设计请参阅 [SDD-Core-Logic.md](docs/architecture/SDD-Core-Logic.md)

### 核心模块

| 模块 | 职责 |
|-------|------|
| **Grid & Entities** | 棋盘和实体数据模型 |
| **GameState** | 游戏状态管理（回合、分数、阶段） |
| **Threat Map** | 威胁范围计算系统 |
| **AI System** | 敌人行为决策逻辑 |
| **Canvas Renderer** | 莫兰迪风格渲染引擎 |
| **Input Handler** | 鼠标/触摸输入处理 |

### 数据流

```
用户输入 → 验证移动 → 更新游戏状态 → 计算敌人威胁 → 渲染画面
                          ↓
                   敌人回合执行 → 检测玩家生死 → 生成新敌人
```

## 🎨 视觉风格

参考微信小游戏《跳一跳》的设计理念：

- **极简低对比度**：避免视觉疲劳
- **莫兰迪色系**：柔和舒适的颜色搭配
- **柔和阴影**：增加立体感但不厚重
- **扁平化几何体**：简洁优雅的视觉语言
- **清晰的信息层级**：威胁、可移动范围一目了然

## 🎯 游戏技巧

1. **优先击杀兵**：兵是经验包，早期容易击杀
2. **注意车辆威胁**：车的威胁范围很大，需要提前规划
3. **利用角落**：边缘位置可以减少敌人的威胁方向
4. **保持距离**：不要让敌人太靠近你
