# 技能系统设计文档 (Skill System Design)

**版本**: v0.2
**日期**: 2026-05-31
**状态**: 与当前代码逻辑一致

---

## 1. 概述

本文档记录当前「回合制象棋肉鸽」原型中已经实现的技能系统。

技能系统是局内成长机制：玩家操控的「帅」通过得分自动获得随机技能，技能可以叠加等级，并在核心回合流程中自动生效。

当前实现以 `src/core/SkillSystem.ts` 和 `src/core/GameLogic.ts` 为准。

### 1.1 设计目标

- 通过随机成长增强单局变化。
- 保持技能效果自动化，减少额外操作入口。
- 用少量技能覆盖生存、位移、控制、阻挡敌人接近和自动清场。
- 让技能等级叠加后直接提升数值表现。

### 1.2 当前范围

当前版本没有实现三选一 Roll、稀有度权重、能量条、主动释放、局前携带或友军召唤。

---

## 2. 技能获取与成长

### 2.1 技能池

当前技能池包含 5 个技能，随机获得时等权抽取。

| 技能 ID | 名称 | 类型 | 主要作用 |
|---|---|---|---|
| `armor` | 铁甲护身 | 生存 | 抵挡致命攻击 |
| `intimidate` | 借刀杀人 | 控制 | 概率冻结相邻敌人 |
| `castling` | 王车易位 | 位移 | 冷却就绪时传送到任意空格 |
| `aura` | 将军威势 | 防线 | 阻止敌人主动走入威慑范围 |
| `siege` | 兵临城下 | 清场 | 定期自动击杀随机小兵 |

### 2.2 触发阈值

技能根据当前分数自动触发：

| 上一次技能分数 | 下一次触发分数 |
|---|---|
| 0 | 3 |
| 3 | 7 |
| 7 及以后 | 上一次技能分数 + 5 |

如果一次得分跨过多个阈值，会连续获得多个随机技能。

### 2.3 等级叠加

每次抽到已有技能时，该技能等级 +1。不同技能的等级效果如下：

| 技能 | 叠加效果 |
|---|---|
| 铁甲护身 | 增加可抵挡次数 |
| 借刀杀人 | 提高冻结概率 |
| 王车易位 | 缩短冷却时间，最低 2 回合 |
| 将军威势 | 扩大威慑范围，最高 2 格 |
| 兵临城下 | 缩短触发间隔，最低每 1 回合 |

## 3. 技能详细设计

### 3.1 铁甲护身 (`armor`)

**效果**: 抵挡一次会导致玩家死亡的攻击。

**触发方式**: 自动触发。当敌人威胁范围命中玩家并将造成死亡时，如果护甲层数大于 0，则消耗 1 层护甲并免死。

**数值**:

| 等级 | 护甲层数 |
|---|---|
| N | N 层 |

**流程位置**: 敌人回合中，完成威胁判定后、进入死亡结算前。

---

### 3.2 借刀杀人 (`intimidate`)

**效果**: 敌人行动前，玩家相邻范围内的敌人有概率被冻结，本回合无法移动。

**触发方式**: 每个敌人回合开始时自动判定。

**判定范围**: 与玩家切比雪夫距离小于等于 1 的敌人，即玩家周围 8 格和同格判定范围。

**数值**:

| 等级 | 冻结概率 |
|---|---|
| 1 | 30% |
| 2 | 45% |
| 3 | 60% |
| 4+ | 75% |

冻结概率公式：

```typescript
Math.min(0.30 + (level - 1) * 0.15, 0.75)
```

**流程位置**: 敌人回合开始时，刷新威胁图前。

---

### 3.3 王车易位 (`castling`)

**效果**: 冷却就绪时，玩家可以无视普通移动距离限制，传送到棋盘上任意空格。

**触发方式**: 自动扩展玩家可移动格。玩家选择距离大于 1 的目标格时，视为使用王车易位并进入冷却。

**可传送目标**:

- 任意空格。
- 玩家当前格。
- 不能直接传送到有敌人的格子。

**数值**:

| 等级 | 冷却 |
|---|---|
| 1 | 4 回合 |
| 2 | 3 回合 |
| 3+ | 2 回合 |

冷却公式：

```typescript
Math.max(4 - level + 1, 2)
```

首次获得王车易位时，冷却为 0，可以立即使用。

**流程位置**:

- 玩家回合开始前刷新可移动范围。
- 玩家移动后，如果移动距离大于 1，则设置冷却。
- 每个敌人回合结束时冷却 -1。

---

### 3.4 将军威势 (`aura`)

**效果**: 敌人移动时不会主动走入玩家周围的威慑范围。

**触发方式**: 敌人准备执行 `nextMove` 时自动检查。如果目标格位于威慑范围内，该敌人本回合不移动。

**数值**:

| 等级 | 威慑范围 |
|---|---|
| 1 | 1 格 |
| 2+ | 2 格 |

范围公式：

```typescript
Math.min(level, 2)
```

**距离算法**: 使用切比雪夫距离，即横向、纵向、斜向都按格数计算。

**流程位置**: 敌人回合中，冻结判定之后、敌人实际移动之前。

---

### 3.5 兵临城下 (`siege`)

**效果**: 定期自动击杀场上一名随机小兵，并获得 1 分。

**触发方式**: 每个敌人回合结算时自动递减计时器。计时器归零后尝试触发。

**目标限制**: 只会选择仍存活的小兵。若场上没有小兵，本次效果跳过，但计时器仍会重置。

**数值**:

| 等级 | 触发间隔 |
|---|---|
| 1 | 3 回合 |
| 2 | 2 回合 |
| 3+ | 1 回合 |

间隔公式：

```typescript
Math.max(3 - level + 1, 1)
```

首次获得兵临城下时，计时器初始化为 3 回合。

**流程位置**: 敌人移动结算后、王车易位冷却递减前。

## 4. 回合流程集成

### 4.1 玩家回合

1. 玩家选择可移动格。
2. 如果目标格有敌人，击杀敌人并获得分数。
3. 根据得分检查是否达到技能阈值。
4. 达到阈值则随机获得技能，可能连续获得多个。
5. 计算敌人下一步移动。
6. 进入敌人回合。

### 4.2 敌人回合

1. 借刀杀人判定相邻敌人冻结。
2. 刷新敌人威胁范围。
3. 判断玩家是否处于敌人威胁范围。
4. 若玩家会死亡，铁甲护身尝试抵挡。
5. 未被冻结的敌人尝试移动。
6. 将军威势阻止敌人走入威慑范围。
7. 兵临城下递减计时并可能击杀小兵。
8. 王车易位冷却递减。
9. 再次根据分数检查技能阈值。
10. 生成新敌人并回到玩家回合。
